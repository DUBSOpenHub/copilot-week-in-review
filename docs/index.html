Features

What you actually get

Most weekly-update tools make you type the input. This one skips the typing.

Two-question intake
The agent asks Source (notes / GitHub pull / both) and Audience(manager / team / yourself). That's it. No flags, no config.
Auto-pulls from GitHub
Pick “pull from my GitHub” and the agent uses gh to gather your last 7 days of PRs, reviews, issues, and comments — grouped by workstream, noise filtered out.
Audience-aware tone
Manager leads with outcomes & impact. Team highlights handoffs & what's next. Yourself keeps your voice & small wins.
Issue-ready output
GitHub-flavored markdown with - [x] checkboxes your team can click as work ships. Paste it straight into an issue, Slack, or email.
No repo required
Runs in any GitHub Copilot app session or Copilot CLI session. No setup, no cloned project, no boilerplate. Just type @week-in-review.
Conversational refinement
Ask for “more casual,” “punchier,” “switch audience to team,” or “open it as an issue in myorg/updates.” Iterate in plain English.

Get started

Install in 30 seconds

1

Install the agent

$curl -fsSL https://raw.githubusercontent.com/DUBSOpenHub/copilot-week-in-review/main/quickstart.sh | bash

Drops one file into ~/.copilot/agents/. No git clone, no config, no API keys.

2

Use it

In either the GitHub Copilot app or a Copilot CLI session, type:

@week-in-review

Answer the two intake questions. Done.

Requirements: GitHub Copilot (subscription) · macOS, Linux, or WSL · the gh CLI for the "pull from GitHub" mode (gh auth login once).

Privacy
Your notes stay in your Copilot session. When you pick the “pull from GitHub” mode, the agent runs gh locally as you — nothing leaves your machine except the GitHub API calls gh already makes on your behalf. No telemetry, no analytics, no third parties.

Tool-light by design: ask_user for the intake, gh (read-only) for the GitHub pull. That's it.
